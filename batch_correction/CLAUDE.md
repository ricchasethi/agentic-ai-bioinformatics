# CLAUDE.md — UK Biobank OLINK Batch Correction Project

## Project overview

Human-in-the-loop agentic pipeline for batch effect detection and correction
of UK Biobank OLINK proteomics data, applied to Alzheimer's Disease biomarker
discovery. Built with LangGraph + Claude claude-sonnet-4-20250514.

A **reflection agent** (`node_reflect_on_assessment`) sits between batch
assessment and strategy proposal. It inspects every confirmed batch field's
η² against severity thresholds, prunes fields that show no meaningful batch
effect, and pre-fills a correction strategy prior — so the strategy LLM and
the human checkpoint both receive a cleaner, already-validated plan.

```

  → node_detect_batch_fields           ← LLM: classifies metadata columns

  → node_reflect_on_assessment         ← ★ reflection agent: prunes/demotes fields
  → node_propose_strategy              ← LLM: refines reflection prior
  ⏸ human_approve_strategy             ← INTERRUPT 2

  → node_validate_correction           ← η² reduction, signal preservation check
  ⏸ human_accept_results               ← INTERRUPT 3

```

Error path: `node_load_data` → `END` (if `final_status == "ERROR"`).
Reject path: `human_accept_results` → `END` (if `final_status == "REJECTED_RETRY"`).

## Critical constraints — read before touching any code

### 1. Never store DataFrames in LangGraph state

`MemorySaver` serialises state to msgpack after every node.
DataFrames, numpy arrays, and numpy scalar types all fail serialisation.

**Rule: state fields must be plain Python primitives only.**
Allowed: `str`, `int`, `float`, `bool`, `list`, `dict` (with primitive values).
Forbidden: `pd.DataFrame`, `pd.Series`, `np.ndarray`, `np.bool_`, `np.int64`,
`np.float64`, and any other numpy/pandas type.

**Pattern for DataFrames:**
```python
# WRONG — crashes at checkpoint
state["df"] = my_dataframe

# CORRECT — save to parquet, store path string
state["df_path"] = _save_df(my_dataframe, "df_name")

# In the next node, load it back
df = _load_df(state["df_path"])
```

**Pattern for numpy scalars in dicts:**
Always call `_sanitise()` on any dict before storing in state:
```python
state["metrics"] = _sanitise(my_metrics_dict)
```

`_sanitise()` recursively converts `np.bool_` → `bool`, `np.integer` → `int`,
`np.floating` → `float`, `np.ndarray` → `list`. It is defined in
`batch_correction_agent.py` and must be called on ALL dicts written to state.

Common traps:
- `df[col].nunique() < 50` → produces `np.bool_` — wrap in `bool()`
- `np.mean(arr) > threshold` → produces `np.bool_` — wrap in `bool()`
- `sum(p < 0.05 for p in pvals)` → safe, `sum()` returns Python `int`
- `any(...)` → safe, always returns Python `bool`

### 2. DataFrame cache helpers

Two helpers defined in `batch_correction_agent.py` — always use them:

```python
CACHE_DIR = "results/batch/cache"

def _save_df(df: pd.DataFrame, name: str) -> str:
    """Saves df to parquet, returns path string for state."""

def _load_df(path: str) -> pd.DataFrame:
    """Loads df from parquet path stored in state."""
```

Use parquet (not CSV) for the cache — it preserves dtypes and is faster
for repeated read/write during a pipeline run.

### 3. QC ordering — do not change this sequence

Strict ordering is required. Each step assumes the previous is complete:

```
1. Missingness filtering     — remove samples/proteins with >20% missing
2. Outlier winsorisation     — clip at ±4 SD per protein
3. Imputation                — median per protein for residual NAs
4. Batch effect detection    — assess η² on clean, complete matrix
5. Reflection                — prune/demote fields with no batch signal
6. Batch correction (ComBat) — correct on pre-INT values
7. Rank-based INT            — normalise AFTER batch correction
8. Association testing       — logistic regression / Cox
```

Never apply INT before batch correction — ComBat models additive/multiplicative
batch structure that INT destroys.

Never batch-correct before imputation — ComBat requires a complete matrix.

### 4. ComBat — always protect biological variables

When calling pyComBat (`inmoose.pycombat.pycombat_norm`), the `covar_mod`
argument must always include `AD_case`, `age`, and `sex` as protected
covariates. Without this, ComBat treats case-control differences as batch
noise and removes the biological signal you are trying to find.

```python
protect_vars = ['age', 'sex', 'AD_case', 'apoe_e4']
covar_df = df[[v for v in protect_vars if v in df.columns]]
corrected = pycombat_norm(protein_matrix.T, batch_labels, covar_mod=covar_df.T)
```

Note: pyComBat expects proteins × samples (transposed relative to pandas
convention). Always pass `.T` and then transpose the result back with `.T`.

### 5. LangGraph interrupt/resume pattern

Human checkpoints use `interrupt()` from `langgraph.types`. The correct
resume pattern is `Command(resume=human_input)` from `langgraph.types`.

The pipeline suspends at nodes listed in `interrupt_before` in `compile()`:
```python
compiled = graph.compile(
    checkpointer=memory,
    interrupt_before=[
        "human_confirm_batch_fields",
        "human_approve_strategy",
        "human_accept_results"
    ]
)
```

The runner loop in `run_pipeline()` uses `_stream_until_interrupt()` which
calls `pipeline.stream(input_val, config, stream_mode="values")` — use
`stream_mode="values"` to get full state snapshots, not update diffs.

### 6. Reflection agent rules

The reflection node (`node_reflect_on_assessment`) runs up to
`_MAX_REFLECT_ROUNDS` (default 3) iterations. Rules it enforces:

- Fields are classified using the same thresholds as §Batch severity thresholds.
- A LOW field is **demoted to covariate** — it goes into regression but NOT
  into ComBat. This prevents over-correction of marginal technical effects.
- A NONE field is **dropped** — it carries no correctable variance.
- If ALL fields are demoted/dropped, the method falls back to `covariate_only`
  or `none`, and `correction_needed` is set to `False`.
- The LLM critiques each round's decisions. If the API is unavailable, the
  automated classification stands without critique.
- The full per-round log is stored in `state["reflection_log"]` and written
  to `audit_trail.json` and `correction_report.md`.
- The reflection node pre-fills `state["proposed_strategy"]` with a validated
  prior. `node_propose_strategy` uses this prior and refines it further;
  it never overwrites the reflection's drop/demote decisions without re-justifying.

**Do not add human interrupts inside the reflection loop.** The loop is
automated and bounded; the human sees the reflection summary at
`human_approve_strategy` (Checkpoint 2).

### 7. State field added by reflection

`reflection_log: list[dict]` — new field in `BatchCorrectionState`.
Each entry is one reflection round:

```python
{
  "round":          int,
  "decisions":      [{"field": str, "action": str, "severity": str, "reasoning": str}],
  "revised_plan":   {
      "confirmed_batch_fields": [...],
      "primary_batch_field":    str,
      "demoted_fields":         [...],
      "dropped_fields":         [...],
      "recommended_method":     str,
      "overall_reasoning":      str,
  },
  "llm_commentary": str,
}
```

Always initialise `reflection_log=[]` in `initial_state` — omitting it will
cause a KeyError at the reflection node.

## Data conventions

### Column naming

Protein NPX columns: any numeric column whose name does not match any of the
`known_meta_patterns` list in `node_load_data`. Add patterns to that list if
your data has additional metadata columns being misclassified as proteins.

Known metadata patterns (checked as substrings, case-insensitive):
`eid`, `age`, `sex`, `bmi`, `smoking`, `education`, `apoe`, `AD_case`,
`AD_status`, `days_to`, `PC`, `131`, `53-`, `plate`, `batch`, `centre`,
`date`, `time`, `quality`, `freeze`, `processing`, `well`, `source`, `collection`

### UK Biobank field IDs used

| Field | Description |
|-------|-------------|
| `53-0.0` | Date of attending assessment centre (blood draw date) |
| `131036` | Date G30 first reported (Alzheimer's disease) |
| `131037` | Source of report of G30 |
| `131032` | Date F00 first reported (dementia in Alzheimer's disease) |
| `131033` | Source of report of F00 |
| `plate_id` | OLINK measurement plate (primary ComBat batch variable) |
| `assessment_centre` | UKB centre code (secondary batch variable) |

### Case definition

AD cases: ICD-10 G30.x or F00.x, diagnosed AFTER blood draw date (incident
cases only). Prevalent cases (diagnosed before blood draw) are excluded from
the primary analysis to avoid reverse causation.

Source reliability: HES inpatient (11), HES outpatient (51), GP (61) are
reliable. Self-report (31) is excluded from primary case definition.

### Batch severity thresholds

These thresholds are used identically by `node_assess_batch_effects` and
`node_reflect_on_assessment`. Do not change one without changing the other.

| η² | % proteins affected | Severity | Reflection action | Correction action |
|----|----|----|----|----|
| ≥ 0.05 | ≥ 30% | HIGH | KEEP | ComBat required |
| ≥ 0.01 | ≥ 10% | MODERATE | KEEP | ComBat recommended |
| ≥ 0.001 | ≥ 2% | LOW | DEMOTE to covariate | Covariate in regression |
| < 0.001 | < 2% | NONE | DROP | Exclude entirely |

## Known issues and decisions

**pyComBat covariate matrix warning** — pyComBat may print:
`[WARNING] The covariate matrix seems to be transposed.`
This is a known pyComBat warning that fires when the covariate matrix
dimensions are non-square. It can be ignored — the computation proceeds
correctly. The warning does not indicate a bug.

**Small batches** — ComBat requires ≥3 samples per batch for variance
estimation. Batches with fewer samples are removed before correction in
`_apply_combat()`. If many batches are small after QC filtering, consider
merging nearby batches by collection date before running ComBat.

**Parquet type preservation** — when reloading from the parquet cache, mixed
integer/float columns may change dtype slightly (e.g. `Int64` → `float64` if
NAs are present). This is expected pandas parquet behaviour and does not
affect downstream analysis.

**Reflection convergence** — the reflection loop exits when the field
classification is stable between consecutive rounds, or after
`_MAX_REFLECT_ROUNDS` (3) rounds. If you see the loop running all three rounds
without converging, check that re-assessment is not introducing noise from a
very small protein sample (default `sample_n=150`).

**Reflection and very small datasets** — if the data has fewer than 200
proteins the reflection node samples all proteins. The η² estimates will be
noisier; consider raising thresholds or increasing `sample_n` accordingly.
